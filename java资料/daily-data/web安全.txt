一：攻击手段

1.xss跨站脚本攻击  
   1）过程：输入框 <script> alert('哈哈哈哈') </script>  ＝> 存储到数据库
   		   另一个页面展示：从数据库中取出value =>  执行恶意脚本
   2）防范：< > ' " 进行转义  jstl c: escapeHtml = 'true' 
2.csrf攻击
   1) 过程： 
   		受害者：C
   		被攻击网站：A
   		攻击者：B
   				用户C登录网站A，输入用户名，密码，获取了cookie，在没有推出网站A的情况下访问网站B，
   				网站B是一个存在xss攻击的站点，此时B向A发起请求，由于cookie存在，A接收了B的请求，。。。
   	2) 防范：cookie设置Httponly属性  增加一层token验证 通过Referer识别请求来源，拒绝非法请求

3.sql注入
　	1）过程  sql语句未启用预编译语法，使用了拼接的sql，此时只要客户端输入 ‘ or '1'='1' '类似语句，就完蛋了
	2）防范  PrepareStatement MyBatis ＃｛name｝  避免数据库明文存储用户密码等敏感信息（哈希加盐）  正确处理系统异常
	
4.文件上传漏洞
	1) 过程：上传了可执行程序或者脚本或木马程序
	2) 防范：不能仅仅通过文件后缀名来判断上传文件的类型
			1）通过魔数来判断文件类型（前28个字节表示文件类型）
			2）对文件名进行重新随机命名
			3）对图片等资源进行相应程度的缩放，破坏可执行程序的结构 ＝> imagemagick高手（缩放，水印，锐化，截取，图像格式转换。。。）
5.DDoS分布式拒绝服务攻击：是目前最为强大，最难以防范的攻击之一 “肉鸡”	SYN Flood（三次握手） DNS Query	CC攻击	  	

二：安全算法
1.摘要算法 原值－> 哈希函数 －> 哈希值（长度固定）
	无论源码多长，输出的哈希值都是固定长度的
	源码相同，哈希值必然相同，源码不同，哈希值一般不同
	不可逆
	MD5（128位）  SHA1（160位，速度更慢，更安全） 16进制编码  Base64编码
	破解方案：彩虹表破解法
2.加密算法
	1）对称性加密
		明文－(密钥)>>密文－(密钥)>>明文（密钥只有一个）
		DES AES（对称加密中最流行的加密算法之一）
	2）非对称性加密（复杂，速度不如对称性加密算法）
		明文－私钥>>密文-公钥>>明文
		原理：两个大素数相乘很容易，反之对其因式分解却很难
3.数字签名
		明文－摘要算法>>摘要串->>私钥－>>数字签名（明文，数字签名传送给数据获取方）
		数据获取方采用相同的摘要算法，得到摘要串，并使用公钥，将数字签名解密，获得摘要串，比较两个摘要串是否一致，进而判断数据是否被篡改
